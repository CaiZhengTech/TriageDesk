## Review outcome and deviations

Ledger: "Task 2: complete (commits 3600a66..680b6b1 on feat/02-db-ingest, review clean/approved;
PR #21 OPEN pending Task 3 — issue #2 spans both)."

Two deviations from the plan's literal snippets, both flagged and accepted in review (PR #21
body):

1. **Dependency-override bug.** The plan's `tests/conftest.py`/test used
   `app.dependency_overrides[get_db] = lambda: iter([test_db])`. Against the installed
   FastAPI/Starlette version, FastAPI only special-cases dependencies whose callable *itself*
   is a generator function (`inspect.isgeneratorfunction`); a lambda that merely returns an
   iterator isn't recognized, so the endpoint received the raw iterator instead of the session
   (`AttributeError: 'list_iterator' object has no attribute 'get'`). Fixed with a small local
   generator function: `def _override_get_db(): yield test_db`. Confirmed still in place in the
   current `tests/integration/test_ticket_roundtrip.py`.
2. **Ruff config additions.** `ignore = ["B008"]` added to `[tool.ruff.lint]` (FastAPI's
   idiomatic `Depends(...)` in argument defaults otherwise gets flagged) and
   `extend-exclude = ["alembic/versions"]` added to `[tool.ruff]` (autogenerated migrations
   aren't meant to conform to project lint style and get regenerated anyway).
